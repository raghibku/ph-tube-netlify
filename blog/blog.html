Discuss the scope of var, let, and const

Scope of var :  var declarations are both globally and locally/function scoped. The variables declared with var can be re-declared and updated which can lead to some problems as one can re-declare the same variable by mistake and instead of showing an error it will give the wrong result.

Scope of let : let is block scoped, so a variable declared in a block with let  is only available for use within that block and can be updated but not re-declared so it doesn't have the similar problem like var.

Scope of const : Can only be accessed within the block they were declared. A variable declared with const remains the same within its scope. It cannot be updated or re-declared.

Tell us the use cases of null and undefined

Use cases of null: 

1. When a variable is declared but not assigned any value, its value is considered to be null. This is is used to initialize variables that will be assigned a value later in the program.

2. When a function does not return any value, it may return null to indicate that no value was returned. This can be useful in cases where the function may not always have a valid value to return.

3.Null values are commonly used in databases to represent missing or unknown data.

4.Null values can be used to represent the absence of a particular property or attribute in an object. 

Use cases of undefined 

1.When working with objects,the optional chaining operator (?.) is used to safely access properties and methods that may be undefined. The optional chaining operator returns undefined instead of throwing an error if any of the intermediate values are undefined.

2.Assigning undefined to uninitialized variables can be helpful for preventing errors.

What do you mean by REST API?

A REST API is simple architectural style for an API which is used for building and interacting with web services. It uses HTTP methods like GET, POST, PUT, DELETE which refers to the reading, updating, creating and deleting of operations concerning resources to enable communication between clients and servers. It uses URLs to represent resources.
